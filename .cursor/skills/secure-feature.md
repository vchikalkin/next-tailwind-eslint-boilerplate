# Навык: Написание безопасных фич и API

Когда я прошу тебя создать форму, обработчик данных или API-роут:
1. Проведи принудительный внутренний аудит безопасности.
2. Всегда используй библиотеку `zod` для валидации входящих данных (как на клиенте, так и на сервере).
3. Используй ранние возвраты (early returns) для обработки ошибок валидации и неверных статус-кодов.
4. Оберни запросы в `try-catch` и возвращай понятные пользователю ошибки на правильном языке (i18n), избегая утечки системных логов в клиентский интерфейс.
